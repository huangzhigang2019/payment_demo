# UI / Screen Contracts: 支付 Demo

**Feature**: 001-payment-demo  
**Purpose**: 约定各界面输入/输出与用户可见行为，便于实现与验收对齐规格。

---

## 1. 主入口（发起交易）

**输入**: 无（进入应用或从结果/设置返回）。  
**用户操作**: 输入交易金额（必填）、可选选择交易类型（若支持）、点击“开始交易”。

**输出/约束**:
- 金额为空、为 0、格式错误或超限时，不发起交易，并提示用户修正（FR-001、Edge Cases）。
- 点击“开始交易”且校验通过后，进入读卡引导界面，并传递金额（及可选类型）。

**可导航**: 设置/信息。

---

## 2. 读卡引导（等待读卡）

**输入**: 交易金额（及可选类型）。  
**用户操作**: 按提示插卡/挥卡/刷磁条；或点击“取消”。

**输出/约束**:
- 读卡成功：进入“处理中”界面，并携带介质类型与卡片信息。
- 读卡超时：提示超时，提供“重试”或“返回”（回到主入口）（FR-004）。
- 用户取消：提示已取消，返回主入口（FR-003）。
- 卡片移开/拔出：提示并允许重试或取消（Edge Cases）。

**可导航**: 仅取消返回主入口；成功/超时/取消后均不滞留本屏。

---

## 3. 处理中（EMV 流程 / 持卡人验证等）

**输入**: 交易金额、介质类型、读卡结果（卡片信息）。  
**用户操作**: 按提示输入 PIN、选择应用或确认等；或取消（若流程支持）。

**输出/约束**:
- 流程正常结束：进入结果页，携带 Processing Result（成功/失败及原因码/摘要）。
- 用户取消或交互超时：进入结果页，状态为“取消”或“超时”，并允许返回主入口（FR-004、FR-003）。
- 异常/不可恢复错误：进入结果页，状态为“失败”，展示可读原因，应用不崩溃（Edge Cases）。

**可导航**: 取消或结束后仅能到结果页，再到主入口。

---

## 4. 结果页

**输入**: 交易金额、交易时间、结果状态（成功/失败/取消/超时）、原因摘要（FR-005）。

**输出/约束**:
- 必须展示：交易金额、交易时间、结果状态、可定位问题的原因摘要。
- 提供“返回”或“再试一笔”等操作，回到主入口；用户在 3 步内回到可再次发起交易（SC-004）。

**可导航**: 返回主入口。

---

## 5. 设置/信息

**输入**: 无。  
**输出/约束**:
- 展示版本信息与关键状态：参数是否已准备、读卡模块是否可用等（FR-006、User Story 3）。

**可导航**: 返回主入口。

---

## 6. 契约与规格对应

| 规格项 | 契约覆盖 |
|--------|----------|
| FR-001 | 主入口金额校验与错误提示 |
| FR-002 | 主入口 → 读卡引导 → 处理中 |
| FR-003 / FR-004 | 读卡引导、处理中的取消与超时 |
| FR-005 | 结果页必显字段与操作 |
| FR-006 | 设置/信息页 |
| FR-008 | 各屏使用 UI_Resource 素材呈现 |

实现时需保证上述输入/输出与导航关系；自动化或手工验收可据此编写用例。
